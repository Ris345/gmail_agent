## Setup (required before running)

**Credentials (gitignored â€” must exist before starting the container):**
1. `credentials.json` in the project root â€” OAuth client secret from Google Cloud Console
2. `gmail/token.json` â€” generated by the OAuth flow; see note below

**OAuth first-run:** Visit `http://localhost:8080/authorize` in your browser after `docker compose up`. The OAuth callback server starts on port 3000 inside the container; Docker's `3000:3000` port mapping forwards the redirect so the browser flow completes normally. The resulting `token.json` is written into the `./gmail` volume mount and persists on the host.

**Ports:** Frontend on `8080` (external) â†’ Flask on `5001` (internal); `3000` is the OAuth redirect URI.

## Architecture

The app is a Flask server that bridges Gmail (via Google API) and an AI agent (via `agno`/OpenAI).

**Request flow:**
- `GET /authorize` â†’ triggers OAuth2 login via `gmail/google_auth.py`, saves token, then calls `read_emails()`
- `GET /retrieveEmails` â†’ calls `gmail/read_emails.py` which reads the last 10 inbox messages using `gmail/token.json`
- `python app.py` also calls `invoke_agent()` at startup â€” **this function is imported but not yet defined in `Agent/parent_agent.py`** (known WIP gap)

**Agent (`Agent/parent_agent.py`):**
- Uses `agno.Agent` backed by GPT-4o
- Has three tools: `check_mail` (GET `/retrieveEmails`), `remove_junk` (POST `/deleteEmails` â€” endpoint not yet implemented in Flask), `trash_cleaner` (stub)
- Scheduled via APScheduler (`BackgroundScheduler`) to run every 3 days automatically
- `Agent/SetInterval/setinterval.py` is an older threading-based interval utility â€” superseded by APScheduler, kept for reference

**Gmail scope:** Currently `gmail.readonly`. Expanding to allow deletion requires changing `SCOPES` in `gmail/google_auth.py` and deleting `token.json` to force re-auth.
